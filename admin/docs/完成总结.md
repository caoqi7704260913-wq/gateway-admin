## 🔑 核心特性

### 1. 与 Gateway 完全兼容

| 特性 | Gateway | Admin | 状态 |
|------|---------|-------|------|
| HMAC Key 格式 | `config:hmac:{app_id}` | `config:hmac:{app_id}` | ✅ 一致 |
| CORS Key 格式 | `config:cors` | `config:cors` | ✅ 一致 |
| 签名算法 | SHA256 + Base64 | SHA256 + Base64 | ✅ 一致 |
| 降级策略 | Redis → Consul → Cache | Redis → Default | ✅ 简化 |

### 2. 完整的注册流程

```
Admin Service                    Redis                     Gateway
     |                             |                          |
     |-- Get HMAC Key ------------>|                          |
     |<-- config:hmac:admin --------|                          |
     |                             |                          |
     |-- Get CORS Config --------->|                          |
     |<-- config:cors -------------|                          |
     |                             |                          |
     |-- Generate Signature ------>|                          |
     |                             |                          |
     |-- POST /register ------------------------------>|
     |                             |                      |
     |<-- 200 OK {service_id} -----|                      |
     |                             |                          |
     |-- POST /heartbeat -------------------------------->|
     |                             |                      |
     |<-- 200 OK --------------------|                      |
```

- ✅ 启动时自动从 Redis 获取配置
- ✅ 如果配置不存在则使用默认值

## ⚠️ 注意事项

### 1. Gateway 必须先启动

Admin 服务依赖 Gateway，启动顺序：
1. Redis
2. Consul (可选)
3. **Gateway**
4. Admin

### 2. Redis 密码必须正确

确保 `.env` 中的 `REDIS_PASSWORD` 与实际密码一致。

### 3. HMAC Key 必须预先创建

Admin 启动前，需要确保 Redis 中有对应的 HMAC Key：

```bash
# 手动创建
redis-cli -a 123123 SET "config:hmac:admin-service" "your-secret-key"

# 或通过 Gateway API 创建
curl -X POST http://localhost:9000/api/config/hmac/key \
  -H "Content-Type: application/json" \
  -d '{"app_id": "admin-service", "secret_key": "your-secret-key"}'
```
