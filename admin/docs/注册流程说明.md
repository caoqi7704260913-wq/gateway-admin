## 🔐 HMAC 签名机制

### 签名生成算法

```python
import hmac
import hashlib
import base64
import time

def generate_hmac_signature(app_id, secret_key, body="", timestamp=None, nonce=None):
    # 1. 准备参数
    if timestamp is None:
        timestamp = str(int(time.time()))
    if nonce is None:
        nonce = hashlib.md5(f"{timestamp}{app_id}".encode()).hexdigest()[:16]
    
    # 2. 构建签名字符串
    message = f"{app_id}\n{timestamp}\n{nonce}\n{body}"
    
    # 3. 生成签名
    signature = hmac.new(
        secret_key.encode(),
        message.encode(),
        hashlib.sha256
    ).digest()
    
    # 4. Base64 编码
    signature_b64 = base64.b64encode(signature).decode()
    
    # 5. 返回头部
    return {
        "X-App-ID": app_id,
        "X-Timestamp": timestamp,
        "X-Nonce": nonce,
        "X-Signature": signature_b64
    }
```

### 使用示例

```python
# 注册时
import json
body = json.dumps(service_info)
headers = generate_hmac_signature("admin-service", secret_key, body=body)
headers["Content-Type"] = "application/json"

    "http://localhost:9000/api/services/register",
    content=body,
    headers=headers
```

## 🚀 启动流程详解

### main.py 中的 lifespan

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 1. 初始化 HTTP 客户端
    await http_client.init()
    
    # 2. 初始化 Redis
    await redis_manager.init()
    
    # 3. 从 Redis 获取并应用默认 CORS 配置
    cors_initialized = await config_service.init_default_cors()
    
    # 4. 注册到 Gateway（包含从 Redis 获取 HMAC Key 和 CORS 配置）
    registered = await register_service.register()
    
    # 5. 启动心跳任务
    heartbeat_task = asyncio.create_task(register_service.heartbeat())
    
    yield
    
    # 关闭时清理
    heartbeat_task.cancel()
    await register_service.unregister()
    await redis_manager.close()
    await http_client.close()
```

## ⚠️ 常见问题

### Q1: 注册失败，返回 401

**原因**: HMAC 签名错误或 Key 不存在

**解决**:
1. 检查 Redis 中是否有 HMAC Key
2. 验证签名生成逻辑
3. 检查时间戳是否在容忍范围内

```bash
# 查看 Redis 中的 HMAC Keys
redis-cli -a 123123 KEYS "config:hmac:*"
```

### Q2: 注册失败，返回 503

**原因**: Gateway 未运行或无法访问

**解决**:
1. 确认 Gateway 正在运行
2. 检查 GATEWAY_URL 配置
3. 检查网络连接

### Q3: CORS 配置未生效

**原因**: Redis 中没有 CORS 配置

**解决**:
1. 运行 `init_default_cors()` 初始化
2. 检查 Redis 连接
3. 验证配置格式

```bash
# 查看 Redis 中的 CORS 配置
redis-cli -a 123123 GET "config:cors"
```

### Q4: 心跳失败

**原因**: 网络问题或服务已注销

**解决**:
1. 检查网络连接
2. 确认 Gateway 仍在运行
3. 查看 Gateway 日志
