## 🎯 各测试脚本详解

### 1. test_user_service.py

**作用**: 模拟一个真实的后端微服务

**功能**:
- ✅ 启动 FastAPI 服务（端口 8001）
- ✅ 自动注册到 Consul
- ✅ 创建 HMAC 密钥
- ✅ 提供用户管理 API
- ✅ 健康检查端点
- ✅ 优雅关闭时自动注销

**API 端点**:
```
GET  /health              - 健康检查
GET  /api/users           - 获取用户列表
GET  /api/users/{id}      - 获取单个用户
POST /api/users           - 创建用户
```

**使用方法**:
```bash
python test_user_service.py
```

**输出示例**:
```
🚀 user-service 启动中...
✅ HMAC 密钥已创建: user-service
✅ 服务已注册到 Consul: user-service (user-service-001)
✅ user-service 已就绪: http://127.0.0.1:8001
```

### 2. test_frontend_simulation.py

**作用**: 模拟前端应用通过 Gateway 访问后端服务

**测试场景**:
1. ✅ 服务注册验证
2. ✅ HMAC 签名认证（正确/错误/过期）
3. ✅ 请求转发（GET/POST）
4. ✅ 负载均衡
5. ✅ 限流功能
6. ✅ 熔断器
7. ✅ 并发请求

**HMAC 签名实现**:
```python
def generate_hmac_signature(app_id, secret_key, body="", timestamp=None, nonce=None):
    """生成符合 Gateway 要求的 HMAC 签名"""
    message = f"{app_id}\n{timestamp}\n{nonce}\n{body}"
    signature = hmac.new(
        secret_key.encode(),
        message.encode(),
        hashlib.sha256
    ).digest()
    
    return {
        "X-App-ID": app_id,
        "X-Timestamp": timestamp,
        "X-Nonce": nonce,
        "X-Signature": base64.b64encode(signature).decode()
    }
```

**使用方法**:
```bash
# 先启动 User Service
python test_user_service.py

# 在新终端运行测试
python test_frontend_simulation.py
```

### 3. test_integration_auto.py

**作用**: 一键自动化测试，无需手动操作

**自动化流程**:
1. 检查 Gateway 是否运行
2. 启动 User Service
3. 等待服务注册到 Consul
4. 运行前端模拟测试
5. 清理资源（停止 User Service）

**优点**:
- ✅ 完全自动化
- ✅ 自动清理资源
- ✅ 适合 CI/CD

**使用方法**:
```bash
python test_integration_auto.py
```

---

### 4. test_gateway_simple.py

**作用**: 测试 Gateway 的核心配置和管理功能

**测试内容**:
- ✅ 健康检查
- ✅ CORS 配置（获取/更新/添加/删除）
- ✅ HMAC 密钥管理（创建/列出）
- ✅ 服务发现
- ✅ 路由 404 处理
- ✅ 限流机制

**特点**:
- ❌ 不需要后端服务
- ❌ 不需要 HMAC 签名（测试时已禁用）
- ✅ 快速验证 Gateway 基础功能

**使用方法**:
```bash
python test_gateway_simple.py
```

---

### 5. run_tests.bat

**作用**: Windows 批处理脚本，一键启动完整测试

**功能**:
1. 检查 Consul 状态
2. 检查 Redis 状态
3. 检查 Gateway 状态
4. 启动 User Service（新窗口）
5. 运行前端模拟测试

**优点**:
- ✅ 双击即可运行
- ✅ 自动检查依赖
- ✅ 友好的中文提示

**使用方法**:
```bash
# 双击运行，或
.\run_tests.bat
```

## 🧪 典型测试场景

### 场景 1: 开发调试

**目标**: 快速验证新功能

**步骤**:
```bash
# 1. 启动 Gateway
python main.py

# 2. 启动 User Service
python test_user_service.py

# 3. 使用 Postman/curl 手动测试
curl -H "X-App-ID: user-service" \
     -H "X-Timestamp: ..." \
     -H "X-Nonce: ..." \
     -H "X-Signature: ..." \
     http://localhost:9000/user/api/users
```

---

### 场景 2: 功能回归测试

**目标**: 确保修改没有破坏现有功能

**步骤**:
```bash
# 运行自动化测试
python test_integration_auto.py

# 或运行简单测试
python test_gateway_simple.py
```

---

### 场景 3: 性能测试

**目标**: 评估 Gateway 的性能瓶颈

**步骤**:
```bash
# 1. 启动所有服务
python test_user_service.py

# 2. 使用 ab 测试
ab -n 10000 -c 100 http://localhost:9000/health

# 3. 使用 wrk 测试
wrk -t12 -c400 -d60s http://localhost:9000/user/api/users
```

---

### 场景 4: 故障演练

**目标**: 测试熔断器和降级机制

**步骤**:
```bash
# 1. 正常运行，发送请求
python test_frontend_simulation.py

# 2. 突然停止 User Service (Ctrl+C)

# 3. 继续发送请求，观察熔断器行为
#    - 应该快速返回 503
#    - 不应该长时间等待超时
```
