## 1. 服务注册与发现

### 模块文件
- `app/services/discovery.py`
- `app/api/routes.py`

### 功能说明
内部服务启动时自动注册到 Gateway，Gateway 维护服务实例列表并提供查询接口。

### 注册流程
```python
POST /api/services/register
{
    "name": "user-service",
    "host": "192.168.1.100",
    "port": 8080,
    "weight": 1,
    "metadata": {"version": "v1"}
}
```

### Redis 存储
```
service:user-service:{instance_id} -> JSON 服务信息
service:healthy:user-service     -> 有序集合 (按权重)
```

### API 端点
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/services/register` | 注册服务 |
| DELETE | `/api/services/{name}` | 注销服务 |
| GET | `/api/services/{name}` | 获取服务列表 |
| GET | `/api/services/{name}/{id}` | 获取指定服务 |

## 2. 健康检查

### 模块文件
- `app/services/health_checker.py`

### 功能说明
Gateway 定时检查已注册服务的健康状态，自动刷新 TTL，不健康的服务自动注销。

### 检查方式
- TCP 连接测试
- HTTP 健康检查（可选）

### 流程
```
启动时 → 扫描所有注册服务 → 定时健康检查
                                    ↓
                    健康 → 刷新 TTL
                    不健康 → 自动注销
```

### 配置
- `REDIS_TTL`: 心跳过期时间（默认 30 秒）
- 健康检查间隔：TTL 的一半

## 3. 负载均衡

### 模块文件
- `app/services/load_balancer.py`

### 支持策略

#### weighted_round_robin（加权轮询）
```python
# 权重越高，被选中的概率越大
services = [
    Service(weight=3),  # 75% 概率
    Service(weight=1),  # 25% 概率
]
```

#### round_robin（轮询）
```python
# 依次选择每个服务
services[0] → services[1] → services[2] → services[0]
```

#### random（随机）
```python
# 随机选择
random.choice(services)
```

### 配置
```env
LOAD_BALANCER_STRATEGY=weighted_round_robin
```

## 4. 请求路由转发

### 模块文件
- `app/services/router.py`
- `app/utils/httpx_manager.py`

### 功能说明
将外部请求透明转发到后端服务，支持请求/响应修改。

### 流程
```
外部请求 → HMAC验证 → 限流 → 负载均衡选择服务 → 转发请求 → 返回响应
```

### 请求修改
- 移除内部不需要的头部
- 添加转发信息

### 超时配置
- `HTTP_TIMEOUT`: 连接/读取超时
- `REQUEST_TIMEOUT`: 请求总超时

## 5. HMAC 签名验证

### 模块文件
- `app/middleware/hmac_middleware.py`
- `app/utils/hmac_validator.py`

### 功能说明
防止请求被篡改，验证请求完整性。

### 签名算法
```
signature = HMAC-SHA256(secret_key, timestamp + nonce + method + path + body)
```

### 请求头
| 头部 | 说明 |
|------|------|
| `X-Signature` | HMAC 签名（Base64 编码） |
| `X-Timestamp` | 时间戳（秒） |
| `X-Nonce` | 随机字符串 |

### 验证流程
```
1. 检查时间戳是否在容差范围内
2. 检查 Nonce 是否重复
3. 验证签名
```

### 排除路径
- `/health`, `/healthz`
- `/api/services/register`
- `/api/services/{name}`
- `/docs`, `/redoc`

### 配置
```env
HMAC_SECRET_KEY=your-key
HMAC_ENABLED=True
HMAC_TIMESTAMP_TOLERANCE=300
```

## 7. 限流

### 模块文件
- `app/middleware/rate_limiter.py`

### 算法
滑动窗口算法，基于 Redis 存储。

### 流程
```
请求 → 检查 key (IP:时间窗口)
           ↓
    超限 → 返回 429
    未超限 → 计数 +1 → 放行
```

### 响应头
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1712659200
```

### 429 响应
```json
{
    "error": "Too Many Requests",
    "message": "请求过于频繁，请 45 秒后重试",
    "retry_after": 45
}
```

### 排除路径
- `/health`
- `/healthz`
- `/docs`
- `/redoc`

### 默认配置
```python
max_requests=100      # 每窗口最大请求数
window_seconds=60     # 窗口大小（秒）
```

## 内部服务接入流程

### 1. 注册服务
```bash
curl -X POST http://gateway:8000/api/services/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "user-service",
    "host": "user-svc",
    "port": 8080,
    "weight": 1
  }'
```

### 2. 获取 HMAC 密钥
```bash
curl -X POST http://gateway:8000/api/config/hmac/key \
  -H "Content-Type: application/json" \
  -d '{"app_id": "user-service"}'
```

### 3. 发送签名请求
```python
import time
import hmac
import hashlib
import httpx

def sign_request(secret_key: str, method: str, path: str, body: str = ""):
    timestamp = str(int(time.time()))
    nonce = "random_string"
    message = f"{timestamp}{nonce}{method}{path}{body}"
    signature = hmac.new(
        secret_key.encode(),
        message.encode(),
        hashlib.sha256
    ).base64()

    return {
        "X-Timestamp": timestamp,
        "X-Nonce": nonce,
        "X-Signature": signature
    }
```

## 9. 熔断器

### 模块文件
- `app/services/circuit_breaker.py`
- `app/services/router.py` (集成)

### 功能说明
保护后端服务，当失败率过高时快速失败，防止级联故障。

### 状态转换

```
CLOSED ──失败次数≥阈值──→ OPEN
  ↑                           │
  │                      超时后
  │                           ↓
  └───成功次数≥阈值──── HALF_OPEN ──失败──→ OPEN
```

### 状态说明

| 状态 | 说明 | 行为 |
|------|------|------|
| CLOSED | 关闭 | 正常请求，失败计数 |
| OPEN | 开启 | 快速失败，不向后端发请求 |
| HALF_OPEN | 半开 | 允许1个请求，通过则关闭，否则继续开启 |

### 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `failure_threshold` | 失败次数阈值 | 5 |
| `success_threshold` | 半开后成功阈值 | 2 |
| `timeout` | 熔断超时（秒） | 30 |

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/circuit-breakers` | 获取所有熔断器状态 |
| GET | `/api/circuit-breakers/{name}` | 获取指定熔断器状态 |
| POST | `/api/circuit-breakers/{name}/reset` | 重置熔断器 |

### 熔断响应

当熔断开启时，返回 503 状态码：
```json
{
    "error": "Service temporarily unavailable",
    "service": "user-service",
    "circuit_state": "open"
}
```
