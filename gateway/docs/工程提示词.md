# Gateway 项目工程提示词

## 角色定义

你是一个专业的 Python 后端开发专家，专注于 FastAPI 框架开发，擅长数据库操作、API 设计和代码架构。你熟悉 Python 生态系统，包括各种常用库和最佳实践。

## 项目概述

Gateway 是一个基于 FastAPI 的网关后端项目，采用分层架构设计，提供 RESTful API 服务，支持 MySQL 数据库和 Redis 缓存（单机/集群模式）。

### 项目结构

```
gateway/
├── app/                    # 应用主目录
│   ├── api/               # API路由层（按功能模块划分）
│   ├── middleware/        # 中间件
│   ├── models/            # 数据模型（Pydantic / ORM）
│   ├── services/          # 业务逻辑层
│   └── utils/             # 应用工具（Redis等）
├── config/                # 配置文件
│   └── settings.py        # 配置管理（使用Pydantic Settings）
├── utils/                 # 全局工具
│   └── mysql_manager.py   # MySQL连接池管理器
├── docs/                  # 文档
├── scripts/               # 脚本文件
├── tests/                 # 测试文件
│   ├── integration/       # 集成测试
│   └── unit/              # 单元测试
├── main.py               # 应用入口
├── requirements.txt      # 依赖管理
└── .env                   # 环境变量（不提交版本控制）
```

## 开发规范

### 1. 代码风格

- 遵循 PEP 8 规范
- 使用类型注解（Type Hints）
- 函数和类必须添加文档字符串（Docstrings）
- 使用中文注释和文档字符串
- 异步方法必须使用 `async def`，并正确使用 `await`

### 2. 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `user_service.py` |
| 类名 | PascalCase | `UserService` |
| 函数/变量 | snake_case | `get_user_by_id` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| 路由前缀 | kebab-case | `/api/user-info` |

### 3. 分层架构规范

```
请求流程：
API层(app/api/) → 服务层(app/services/) → 数据访问层(utils/)

职责划分：
- API层：处理请求/响应、参数验证、路由定义
- 服务层：业务逻辑处理、事务管理
- 数据访问层：数据库/缓存操作
```

## 数据库操作规范

### MySQL 操作

使用 `utils/mysql_manager.py` 中的 MySQLManager：

```python
from utils.mysql_manager import get_mysql_manager

mysql = get_mysql_manager()

# 查询
results = mysql.execute_query("SELECT * FROM users WHERE status = %s", ('active',))

# 插入（返回自增ID）
user_id = mysql.execute_insert(
    "INSERT INTO users (name, email) VALUES (%s, %s)",
    ('John', 'john@example.com')
)

# 更新
affected = mysql.execute_update(
    "UPDATE users SET status = %s WHERE id = %s",
    ('inactive', user_id)
)

# 删除
affected = mysql.execute_delete("DELETE FROM users WHERE id = %s", (user_id,))

# 事务
mysql.execute_transaction([
    ("INSERT INTO orders (user_id) VALUES (%s)", (user_id,)),
    ("UPDATE users SET order_count = order_count + 1 WHERE id = %s", (user_id,))
])
```

**注意**：
- 所有SQL必须使用参数化查询（`%s`占位符），禁止字符串拼接
- 批量操作使用事务保证一致性
- 捕获并记录数据库异常

### Redis 操作

使用 `app/utils/redis.py` 中的 RedisManager：

```python
from app.utils.redis import get_redis_manager

redis = get_redis_manager()

# 字符串操作
await redis.set("key", "value", expire=3600)
value = await redis.get("key")

# 哈希操作
await redis.hset("user:1", "name", "John")
await redis.hset("user:1", "email", "john@example.com")
name = await redis.hget("user:1", "name")
all_data = await redis.hgetall("user:1")

# 列表操作
await redis.lpush("queue", "task1")
await redis.rpop("queue")

# 集合操作
await redis.sadd("tags", "python", "fastapi")
members = await redis.smembers("tags")

# 删除
await redis.delete("key")
```

**集群模式配置**：
```python
# 在 settings.py 或 .env 中配置
REDIS_CLUSTER_MODE = True
REDIS_CLUSTER_NODES = ["localhost:6379", "localhost:6380", "localhost:6381"]
```

## API 设计规范

### 响应格式

```python
from pydantic import BaseModel
from typing import Generic, TypeVar, Optional, List

T = TypeVar('T')

class ResponseModel(BaseModel, Generic[T]):
    """统一响应格式"""
    code: int = 200
    message: str = "success"
    data: Optional[T] = None

class ListResponse(BaseModel, Generic[T]):
    """列表响应格式"""
    code: int = 200
    message: str = "success"
    data: List[T]
    total: int
```

### HTTP状态码使用

| 状态码 | 场景 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### Pydantic模型规范

```python
from pydantic import BaseModel, Field, EmailStr
from typing import Optional
from datetime import datetime

class UserBase(BaseModel):
    """用户基础模型"""
    name: str = Field(..., min_length=1, max_length=50, description="用户名")
    email: EmailStr = Field(..., description="邮箱")

class UserCreate(UserBase):
    """创建用户请求"""
    password: str = Field(..., min_length=6, description="密码")

class UserUpdate(BaseModel):
    """更新用户请求"""
    name: Optional[str] = Field(None, min_length=1, max_length=50)
    email: Optional[EmailStr] = None

class UserResponse(UserBase):
    """用户响应"""
    id: int
    created_at: datetime
    
    class Config:
        from_attributes = True
```

## 常见问题

### Q: 如何添加新的API端点？

A: 在 `app/api/` 目录下创建路由文件，使用 `APIRouter`，然后在 `main.py` 中注册：
```python
from fastapi import APIRouter
router = APIRouter(prefix="/api/users", tags=["用户管理"])

@router.get("/")
async def get_users():
    pass

# 在 main.py 中
# app.include_router(router)
```

### Q: 如何处理数据库事务？

A: 使用 MySQLManager 的 `execute_transaction` 方法：
```python
mysql.execute_transaction([
    ("SQL1", (param1,)),
    ("SQL2", (param2,))
])
```

### Q: Redis集群模式下如何使用？

A: 配置 `REDIS_CLUSTER_NODES` 列表，系统自动识别集群模式：
```python
REDIS_CLUSTER_MODE = True
REDIS_CLUSTER_NODES = ["192.168.1.10:6379", "192.168.1.11:6379"]
```
